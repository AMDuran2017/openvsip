6.8. Diagnostics functions

Unless otherwise noted, all types described in this section are declared in the vsip_csl namespace.

6.8.1. assign_diagnostics

Description. 

template <typename V1, typename V2> 
void assign_diagnostics(V1 lhs, V2 rhs);

Description. This function prints information about available backends for the assignment lhs = rhs;.

Function parameters. 

lhs

The lefthand-side view of the assignment.

rhs

The righthand-side view of the assignment. This typically is an expression, such as A * B or fft(C).

Example. 

Vector<> a;
Vector<> b;
Vector<> c;
...
a = b * c;
assign_diagnostics(a, B * c);

This may generate output similar to this:

--------------------------------------------------------
assign diagnostics:                                            
  dim          : 1  (8)
  LHS    : ...
  RHS    : ...
  is_illegal   : false
  is_rhs_expr  : true
  is_rhs_simple: false
  is_rhs_reorg : true
  is_lhs_split : false
  is_rhs_split : false
  lhs_cost     : 0
  rhs_cost     : 2
  type         : serial_expr
--------------------------------------------------------
  lhs expr: ...
  rhs expr: ...
  -                 user  ct: false  rt: false
  -           dense_expr  ct: false  rt: false
            ...
            ...
  -          loop_fusion  ct:  true  rt:  true  (Expr_Loop)
--------------------------------------------------------

Of particular interest here is the second block of the output, which lists all backends, together with information about their availability. ct: false states that the evaluator was at compile-time determined not to match, while rt: true indicates the runtime validity check passed. Thus, available backends require both ct as well as rt checks to pass, to be selected, and the first match in the list will be dispatched to.
